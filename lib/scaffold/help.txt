Available Commands:

- grunt scaffold:help ->
  Description: Shows this information.
  
- grunt scaffold:create[:templateName] --l=[layer] --test=true ->
  Description:
      Creates directory structure and templates for a specific layer indicated in the --l parameter, with a name [:"templateName"].
      i.e usage: grunt scaffold:create:mymodel --l=model (This will generate templates and folder structure for a model called "mymodel").
      
      Possible values for --l: [model|controller|service|view|full].
      i.e: grunt scaffold:create:MyAwesomeService --l=service
      
      Notice that there is an special option available called "full" to pass to the --l parameter. That's going to create a model, controller, service and the view
      "all at once" for each tier and using the "templateName" specified.
      i.e usage: grunt scaffold:create:cart --l=full
      
      Also, there is an additional attribute --test=true. This will generate templates for "unit testing" for each layer (This can be used in combination with the option "full" in the --l parameter).
      i.e usage: grunt scaffold:create:cart --l=model --test=true
      
- grunt scaffold:remove[:templateName] --l=[layer] --test=true ->
  Description:
      Remove directory structure and templates previously generated by scaffold:create command described above.
      This command uses same parameter as the create one, but for "removing". This applies either, on specific layers or with the use of "full". If --test=true is specified, the code will try to remove the test files as well.
